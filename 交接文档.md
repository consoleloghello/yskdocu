# 公用工程题库 SPA — 项目交接文档

## 一、项目概述

将「公用工程题库」两个 docx 文件解析为结构化 JSON，构建移动端友好的单页面刷题应用（SPA），部署到 GitHub Pages。题库内容为化工企业公用工程岗位的考试题库，涉及火炬、给水加压泵站、罐区、锅炉等 9 个专业系统。

## 四、数据解析（scripts/parse_docx.py）

### 功能
读取 docx 文件，自动识别章节、题型、题目内容、选项、标准答案，输出结构化 JSON。

### 解析逻辑
- **章节检测**：通过已知章节名列表匹配独立段落（"火炬"、"给水加压泵站"等）
- **题型检测**：匹配「一、选择题」「二、填空题」「三、判断题」「四、简答题」「五、实操分析题」等标题
- **选择题**：题干提取答案标记（如 `（B）`），下一行/同行提取 A/B/C/D 选项
- **判断题**：从 `（√）` / `（×）` 标记中提取答案
- **填空题**：每个段落视为一个填空知识点，答案即为题干本身
- **简答题/实操题/应急处理题**：题干行 + 后续「答案：」开头的回答，支持多段答案连续性判断（通过长度、是否锚点行、是否以问号结尾等启发式规则）

### 运行方式
```bash
python scripts/parse_docx.py
```
自动读取两个 docx 文件，生成 `data/外操版.json` 和 `data/内操版.json`。

### 已知数据问题
- 外操版 火炬 的第一组选择题实际包含 10 题（外操岗位）+ 若干扩展题，解析后按题型分组可能有 2 组选择题（13+10），前端已通过展平处理，不影响展示
- 部分简答题的多段答案合并逻辑偶尔会漏接一段（如答案中以「答：」开头而非「答案：」的变体）
- 外操版 docx 中「第二部分 内操岗位题库」标记被解析器跳过，其中包含的内容分布在各个章节中

## 五、前端 SPA（index.html + css/style.css + js/app.js）

### 核心功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 双题库切换（外操版/内操版） | ✅ | 顶部按钮切换，切换时清空错题本 |
| 章节筛选 | ✅ | 横向滚动 chip 列表，点击筛选 |
| 题型二次筛选 | ✅ | 选择题/填空题/判断题/简答题/实操题/应急处理题 |
| 搜索（全文匹配） | ✅ | 匹配题目+答案+章节名+题型，搜索后题型筛选在结果内二次过滤 |
| 全部题目纵向展开 | ✅ | 选中章节+题型后，所有题目以卡片列表形式展示 |
| 选择题点击选项对答案 | ✅ | 点击选项立即反馈正确/错误，高亮正确答案 |
| 显示/隐藏答案 | ✅ | 每题独立控制，支持全部显示/全部隐藏 |
| 错题本 | ✅ | 自动记录错题，支持错题集中复习 |
| 随机打乱 | ✅ | 当前题目列表随机排序 |
| 学习统计 | ✅ | 总题数/错题数/已显示答案数 |
| 状态持久化 | ✅ | 章节、题型、错题本、已显示答案均存 localStorage |
| 入口版本选择遮罩（毛玻璃） | ✅ | 进站弹出，选择外操版/内操版后进入 |
| 响应式布局 | ✅ | 手机优先，适配 320px+ 屏幕 |

### 核心数据流

```
JSON 数据加载 → buildFlat() 展平为 flatQs 数组
  → getCurrentQs() 根据 state 过滤（章节+题型+搜索+错题模式）
    → renderCards() 渲染卡片列表
      → 每题独立「显示答案」控制（revealed Set）
      → 错题自动记录（state.wrongBook）
        → localStorage 持久化
```

### 状态管理（localStorage）

| key | 内容 |
|-----|------|
| `ysk_state` | 当前版本、章节、题型、搜索词、模式、错题本、统计 |
| `ysk_revealed` | 已显示答案的题目 ID 数组 |

### 关键函数说明

| 函数 | 位置 | 作用 |
|------|------|------|
| `loadData()` | app.js | 异步加载 JSON → 构建展平列表 → 渲染 |
| `getCurrentQs()` | app.js | 根据 state 计算当前应显示的题目列表 |
| `renderCards(qs)` | app.js | 接收题目数组，渲染为卡片列表，绑定事件 |
| `renderChapters()` | app.js | 渲染章节 chip 列表 |
| `renderTypeFilters()` | app.js | 渲染题型筛选按钮（带计数） |
| `updateTopActions()` | app.js | 更新顶部按钮状态（错题数、显示/隐藏切换） |

### P1 - 影响使用

#### 已修复

- ✅ ~~版本切换按钮事件丢失~~ 已恢复头部按钮点击事件

#### 仍需处理
- 内操版 给水加压泵站 的扩展题解析后只有 22 题（包含外操岗位 15+15+14+3=47 题和扩展题 4+6+4+5+3=22 题），但从 JSON 读取时同名题型分组合并导致显示异常。实际数据是完整的，需检查 app.js 的展平逻辑是否正确处理同名题型
- 搜索时只匹配了题目+答案+章节+类型，未高亮展示搜索匹配位置

### P2 - 体验优化
- 暂无夜间模式
- 暂无收藏/标记功能
- 暂无答题历史/正确率曲线
- 首页欢迎语显示浏览模式，可根据实际状态动态化
- 切换版本时错题本完全清空（当前是设计如此，但可优化为每个版本独立错题本）

### P3 - 技术债
- 解析器对 docx 的依赖是硬编码的章节名列表，若源文档新增章节需更新 `KNOWN_CHAPTERS`
- 简答题答案连续性判断的启发式规则（`looks_like_new_question()`）不够精确，极端情况下可能把答案正文误判为新题目
- 前端在当前 `renderCards()` 每次都重新渲染全部卡片，大量题目时（如全部 372 题）可能稍慢，可考虑虚拟滚动
- 未添加 Service Worker / PWA 离线支持
- 进入遮罩每次刷新都显示，未添加「记住选择」选项（当前设计如此）
