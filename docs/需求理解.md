 ## 2026-06-19 14:00
 
 ## 一、需求概述
 
 将「公用工程题库」docx 文档中的题库内容解析后，构建为一个单页面应用（SPA），部署到 GitHub Pages，实现随时随地通过手机浏览器刷题备考的目的。
 
 题库内容涉及 9 大专业系统章节（火炬、给水加压泵站、罐区、锅炉、空压站、污水预处理、循环水站、制冷站、制氮站）及新增题库，共计约 500+ 道题，题型涵盖选择题、填空题、判断题、简答题/实操题。题库分为「内操版」和「外操版」两个版本。
 
 ## 二、功能需求
 
 ### P0 - 必须实现
 - [ ] docx 题库解析：程序化读取 `公用工程题库（内操版）.docx` 和 `公用工程题库（外操版）.docx`，识别章节、题型、题目内容、选项及标准答案，输出结构化 JSON 数据文件
 - [ ] 题库浏览：按章节（火炬、给水加压泵站、罐区等）筛选题目
 - [ ] 刷题模式：展示题目 → 用户选择/输入答案 → 显示正确答案与解析
 - [ ] 版本切换：支持「内操版」/「外操版」两个题库版本一键切换
 - [ ] 移动端适配：响应式布局，在手机浏览器上流畅操作
 - [ ] 静态部署：产出为纯静态 HTML + JS + CSS + JSON，零后端依赖，可直接部署到 GitHub Pages
 
 ### P1 - 重要功能
 - [ ] 错题本：错题自动记录（localStorage），支持重新练习
 - [ ] 随机刷题：从当前章节/全部题目中随机抽取
 - [ ] 答题统计：显示答题数量、正确率、进度等
 - [ ] 题型筛选：按选择题/填空题/判断题/简答题筛选
 
 ### P2 - 可选功能
 - [ ] 收藏/标记：收藏重点题目，单独复习
 - [ ] 答题历史：查看每次练习的答题记录
 - [ ] 夜间模式
 
 ## 三、非功能需求
 
 - **性能**：页面首次加载后，题库 JSON 从本地（GitHub Pages）加载，切换无延迟；页面体积控制在 2MB 以内，确保手机流量友好
 - **可用性**：所有状态（当前版本、章节、答题进度、错题本）存储在 localStorage，刷新不丢失
 - **兼容性**：支持现代移动浏览器（Chrome/Safari/微信内置浏览器）
 - **离线可用**：PWA 或 Service Worker 缓存，支持离线刷题（P2）
 
 ## 四、边界与约束
 
 - **技术栈**：纯前端（HTML5 + CSS3 + Vanilla JS 或 React SPA），零后端、零数据库
 - **部署方式**：GitHub Pages 静态托管
 - **不包含**：用户登录、多用户同步、服务端 API、数据分析后台
 - **数据来源**：仅依赖本地 docx 解析产出的 JSON 文件，不依赖外部接口
 - **内容版权**：题库内容仅供个人/企业内部学习使用
 
 ## 五、验收标准
 
 ### 功能点1：docx 题库解析
 - Given 输入 docx 文件，When 运行解析脚本，Then 输出结构完整的 JSON 文件，包含章节、题型、题号、题目内容、选项（如有）、标准答案
 
 ### 功能点2：题库浏览
 - Given 用户打开应用，When 选择章节，Then 仅显示该章节的题目列表
 - Given 用户打开应用，When 切换版本（内操/外操），Then 题库内容随之切换
 
 ### 功能点3：刷题模式
 - Given 用户进入刷题模式，When 点击选项/填写答案，Then 即时反馈正确/错误并显示正确答案
 - Given 用户做错题目，When 查看结果，Then 该题自动记入错题本
 
 ### 功能点4：移动端适配
 - Given 用户在手机上打开应用，When 浏览任何页面，Then 布局紧凑、按钮可点、文字可读
