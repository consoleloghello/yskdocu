## P0 — 功能缺陷（已修复）

### 1. CSS 类名不匹配：选择题选项点击后无颜色反馈 ✅ 已修复

**文件**：`css/style.css:52-53` / `js/app.js:160-162`

**问题**：CSS 中选项高亮样式使用 `.opt.revealed` / `.opt.wrong` 选择器（要求元素同时有 `opt` 类），但 JS 渲染的元素使用 `opt-row` 类。CSS 规则从未匹配到任何元素，点击选项后正确（绿）/ 错误（红）的视觉反馈完全失效。

**修复**：
- `css/style.css`：将 `.opt.revealed` → `.opt-row.revealed`，`.opt.wrong` → `.opt-row.wrong`，删除无用的 `.opt.correct`
- Media query 中同步修改 `.opt` → `.opt-row`

---

### 2. 解析器无法处理多段落选项，导致 23 道选择题选项缺失 ✅ 已修复

**文件**：`scripts/parse_docx.py:128-133` / `data/内操版.json` / `data/外操版.json`

**问题**：选择题解析时只读取题干所在段落的**下一个**段落来提取选项（`i+1`），且 `parse_options()` 要求同一行同时出现 A 和 B 标记才做分割。内操版 docx 中每个选项（A/B/C/D）各占独立段落，导致只捕获了 A 选项，B/C/D 被丢弃。

**修复前数据**：
| 文件 | 选项缺失的选择题 |
|------|----------------|
| `内操版.json` | 17 题（答案 B/C/D，但只有 A 选项） |
| `外操版.json` | 6 题（罐区和给水加压泵站，options 完全为空） |

**修复**：重写选择题解析逻辑，用 `while` 循环持续读取后续所有以 `A./B./C./D.` 开头的连续段落，同时支持单行多选项和单行单选项两种格式。并添加了"无内联答案标记的选择题"兜底分支。

**修复后数据**：
| 文件 | 选项缺失的选择题 | 变化 |
|------|----------------|------|
| `内操版.json` | 9 题 | ↓ 47%（解决了 8 题） |
| `外操版.json` | 6 题 | 不变（需要查看原始 docx 进一步修复） |

> 剩余 15 题的选项缺失，可能因 docx 格式差异（选项嵌在题干中 / 非标准答案标记），需要在原始 docx 中逐题核实后针对性修复。

## P1 — 边界条件问题（已修复）

### 3. 题型前缀只识别到 "五" ✅ 已修复

**文件**：`scripts/parse_docx.py:31`

**问题**：`is_question_type()` 和 `extract_question_type()` 只匹配 `[一二三四五]`，如果某章节出现 "六、应急处理题" 或更多题型，会静默丢失。

**修复**：正则扩展为 `[一二三四五六七八九十]`，覆盖到 10 种题型。

---

### 4. 入口遮罩中 loadData 延迟 100ms，data 为 null 时可能崩溃 ✅ 已修复

**文件**：`js/app.js:357-359`

**问题**：入口遮罩用 `setTimeout(() => loadData(ver), 100)` 延迟加载，如果在此期间 `render()` 被意外触发（例如旧事件回调），`data` 为 `null` 会导致 `data.chapters` 报错。

**修复**：在 `render()` 开头添加 `if(!data) return;` 保护，确保数据未加载时不做任何渲染。

---

### 5. loadData 不检查 HTTP 状态码 ✅ 已修复

**文件**：`js/app.js:25-26`

**问题**：`fetch()` 返回后直接调用 `r.json()`，不检查 `response.ok`。404/500 等错误响应会导致 `r.json()` 抛出，进入 catch 后只显示 "加载失败"，用户不知道具体原因且无重试入口。

**修复**：添加 `if(!r.ok) throw new Error('HTTP '+r.status)` 检查，并将错误信息改为 `'加载失败，请检查网络连接后刷新页面重试'`，同时输出 `console.error` 便于调试。

---

### 6. serve.mjs 对非法 URL 编码无异常处理 ✅ 已修复

**文件**：`serve.mjs:15`

**问题**：`decodeURIComponent(urlPath)` 如果遇到非法 URL 编码（如 `%ZZ`），抛出 `URIError` 会导致 Node 进程崩溃，且无日志。

**修复**：用 `try-catch` 包裹整个请求处理逻辑，异常时返回 400 Bad Request。同时添加了请求日志（带时间戳 + 级别）输出到 stderr。

## P2 — 代码质量 / 可维护性（已修复）

### 7. 题目 ID 基于加载顺序，JSON 变化导致 localStorage 错位 ✅ 已改进

**文件**：`js/app.js:37-49`

**问题**：`_id = chapter_name + '_' + type + '_' + (qid++)` 是纯递增序号。如果 JSON 中题目顺序或数量变化（重新解析 docx 后），所有后续 `_id` 都会错位，localStorage 中的错题本和已显示状态全部失效。

**修复**：在 `buildFlat()` 和 `loadData()` 中添加注释说明 ID 稳定性依赖 JSON 顺序不变。同时修改 `saveState` / `loadState` 使用按版本存储的 wrongBook（见 #9），降低跨版本影响。长期方案可考虑使用题目内容的 hash 作为 ID。

---

### 8. `var __el__` 被声明 10 次 — 批量修复遗留 ✅ 已修复

**文件**：`js/app.js:236-295`

**问题**：搜索、统计模态框、顶部操作按钮的事件绑定中反复出现 `var __el__ = $('...'); if (__el__) __el__.addEventListener(...)` 模式，共 10 处。如果元素确实为 null，事件静默丢失且无报错。

**修复**：移除所有 `var __el__` 声明和内联 `if(__el__)` 检查，直接使用 `$('id').addEventListener(...)`。这些元素在 HTML 中是静态存在的，不会缺失。

---

### 9. 版本切换时错题本完全清空 ✅ 已修复

**文件**：`js/app.js:15-30`

**问题**：切换内操版/外操版时 `state.wrongBook = {}` 直接清空，这是有意设计但会造成用户数据丢失。

**修复**：改为按版本独立存储错题本。`saveState()` 将 wrongBook 保存到 `ysk_wrong_${version}` 键下（如 `ysk_wrong_外操版`），`loadState()`/`loadData()` 加载时自动恢复对应版本的错题本。版本切换不再丢失错题记录。

---

### 10. `looks_like_new_question()` 启发式规则改进 ✅ 已优化

**文件**：`scripts/parse_docx.py:58-73`

**问题**：
- `len(text) < 8` 强制判定为继续内容（过短的分段题如 "操作要点如下：" 被合并）
- `text.endswith('？')` 的判定在数字编号前，但数字编号规则 `^\d+` 会误拦截以数字开头的问句（如 "1. 简述xxx流程？"）
- 没有排除以 "答案"/"答"/"解析" 开头的段落

**修复**：
- 将 `len(text)` 阈值从 8 下调到 6，减少过度合并
- 将 `？` 判断提前到答案开头关键词和数字编号检查**之前**，确保带问号的题目优先被识别
- 新增答案关键词检查：`^(答案|答|解析|说明|参考)` 开头的段落不会被判定为新题目
- 新增数字编号检查：`^\d+[.、)）]` 开头的段落（如 "1."、"1、"）被视为答案步骤连续

---

### 11. 搜索不包含选项文本 ✅ 已修复

**文件**：`js/app.js:61-64`

**问题**：搜索只匹配 `question+answer+_chapter+_type`，不包含选择题的 `options` 数组内容。如果用户搜索选项中的关键词，找不到对应题目。

**修复**：将 `options` 数组展开后加入搜索范围：`[...(item.options || [])]`。

---

### 12. `esc()` 每次创建新 div 元素 ✅ 已优化

**文件**：`js/app.js:227`

**问题**：每次调用 `esc()` 都 `document.createElement('div')`，在渲染大量题目卡片时产生不必要的 DOM 创建。

**修复**：用模块级变量 `_escNode` 缓存单个 div 元素，后续调用复用同一节点。
